# EVOLUTION.md

## 1. 你如何实现提示功能？
在 Sudoku 类中实现了三个核心方法：
- getCandidates(row, col)：获取格子所有合法候选数
- getNextHint()：自动寻找唯一可填数字，作为下一步提示
- hasConflict()：检查棋盘是否冲突

Game 类统一对外提供提示接口。

## 2. 提示功能更属于 Sudoku 还是 Game？
属于 **Sudoku**。
因为提示依赖数独规则、棋盘状态、行列宫限制，这些都是数独本身的属性。
Game 只负责调用，不负责计算。

## 3. 你如何实现探索模式？
探索模式基于：
- 深拷贝主棋盘，创建独立探索棋盘
- 探索过程只修改副本
- 拥有独立历史栈
- 提交时覆盖主棋盘
- 放弃时直接丢弃副本

## 4. 主局面与探索局面的关系？
- 复制关系（深拷贝）
- 完全独立，不共享对象
- 无引用污染
- 提交 = 覆盖
- 放弃 = 销毁

## 5. history 结构是否变化？
没有破坏原有线性 history。
新增了**独立的探索历史栈**，提交后才合并到主历史。

## 6. Homework 1 哪些设计暴露局限？
- 没有深拷贝方法，无法安全做探索
- 没有棋盘状态查询接口
- 无法区分主操作与临时操作
- Undo/Redo 不能支持分支操作

- 提前实现 clone()
- 让 Sudoku 提供更多状态查询能力
- 让 Game 支持“临时会话”模式
- 让历史栈支持隔离操作
